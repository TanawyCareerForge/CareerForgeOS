# Contributing to CareerForge OS

**Version:** 0.1.0-alpha  
**Status:** Draft  
**Purpose:** Explain how contributors propose, specify, implement, test, and document changes.

## Responsibilities

- Follow Specification Driven Development.
- Keep documentation, tests, examples, and changelog entries synchronized.
- Preserve truth-first, explainable, human-first behavior.

## Dependencies

- Governance framework
- Branch strategy
- Pull request template
- Code of conduct

## Inputs

- Issues
- PRDs
- ADRs
- RFCs
- Tests
- Community feedback

## Outputs

- Pull requests
- Reviewed specifications
- Validated implementation changes

## Validation Rules

- Features must start with a PRD, ADR, and RFC unless explicitly classified as maintenance.
- Pull requests must include tests or a documented reason tests are not applicable.
- AI behavior changes must document explainability and truthfulness impact.

## Examples

- A new resume scoring engine requires a PRD, architecture decision, RFC, tests, examples, and documentation.

## Test Cases

- Verify PRs reference relevant specifications.
- Verify changelog updates are present for user-visible changes.
