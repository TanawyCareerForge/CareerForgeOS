# CareerForge OS Roadmap

**Version:** 0.1.0-alpha  
**Status:** Draft  
**Purpose:** Define planned delivery milestones.

## Responsibilities

- Sequence work into specification-driven sprints.
- Communicate scope and dependencies.
- Keep implementation aligned with governance and standards.

## Dependencies

- Vision
- Mission
- Architecture
- Governance framework

## Inputs

- Product vision
- Engineering dependencies
- Community feedback

## Outputs

- Sprint plans
- Release targets
- Prioritized capability map

## Validation Rules

- Every implementation sprint requires PRD, ADR, and RFC coverage.
- Roadmap changes must update the changelog when released.

## Milestones

| Sprint | Focus | Outputs |
| --- | --- | --- |
| 0 | Bootstrap | Governance, standards, repository structure |
| 1 | Career Kernel | Kernel RFC, contracts, validation model |
| 2 | Career Ontology | Entity definitions and JSON Schemas |
| 3 | Resume Intelligence | Resume engine specifications |
| 4 | ATS and Recruiter Intelligence | Quality gates and scoring specifications |
| 5 | Agents and SDK | Agent protocol, SDK contracts, examples |

## Examples

- Sprint 1 cannot implement kernel runtime without a kernel PRD, ADR, and RFC.

## Test Cases

- Verify each sprint has explicit deliverables.
- Verify release plans include documentation updates.
