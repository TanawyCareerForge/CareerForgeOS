# RFC-000 Governance

**Version:** 0.1.0-alpha  
**Status:** Draft  
**Purpose:** Define the RFC process and numbering framework for CareerForge OS.

## Responsibilities

- Standardize protocol and contract proposals.
- Capture rationale, alternatives, and compatibility impact.
- Provide durable specification records.

## Dependencies

- Repository governance
- ADR framework
- PRD framework

## Inputs

- Problem statement
- Proposed contract or protocol
- Compatibility analysis
- Security and explainability considerations

## Outputs

- Accepted, rejected, superseded, or draft RFC
- Implementation checklist
- Test expectations

## Validation Rules

- RFCs must use assigned number ranges.
- RFCs must include purpose, responsibilities, dependencies, inputs, outputs, validation rules, examples, test cases, and revision history.

## RFC Numbering

- RFC-000 Governance
- RFC-100 Kernel
- RFC-200 Ontology
- RFC-300 Knowledge
- RFC-400 Generation
- RFC-500 Optimization
- RFC-600 Evaluation
- RFC-700 Agents
- RFC-800 SDK
- RFC-900 Enterprise

## Examples

- RFC-100 defines Career Kernel control-plane contracts.

## Test Cases

- Verify RFC metadata is complete.
- Verify number range matches topic.
